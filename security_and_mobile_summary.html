Project Executive Summary

We have performed a full security auditing and environments layout refactoring for De-expense Tracker. By patching dynamic rendering loops against Stored XSS, configuring server-free Content Security Policy meta directives, structuring local environment boilerplate, and crafting an Apple-compliant bottom-navigation layout, the platform operates as a secure, fast, and installable Progressive Web App (PWA).

12-Point Security Checklist Compliance

✓

1. Secrets & Environment Isolation

Boilerplate setup in .env.example with Vite prefixed VITE_ tokens; zero hardcoded frontend keys.

✓

2. Cross-Site Scripting (XSS) Prevention

Added dynamic text-encoding escapeHTML helper; escapes user category strings rendered via dangerouslySetInnerHTML.

✓

3. Content Security Policy (CSP)

Strict header integration whitelisting Google Font CDNs (fonts.googleapis.com & fonts.gstatic.com) under client boundaries.

✓

4. Git Credentials Protection

Structured global .gitignore blocking IDE garbage, localized variables .env*, and production bundles.

i

5. Backend Security Metrics (Auth, SQL, Rate Limits, Headers)

Analyzed as Not Applicable (N/A) due to client-side localStorage sandbox. Security is maintained by browser sandbox boundaries.

Apple-style Mobile UI/UX & Navigation

To give mobile users a high-fidelity native app feel, we replaced the cluttered desktop header items with a custom Bottom Navigation Tab Bar using Framer Motion.

  • Bottom Tab Navigation: Fixed bottom bar with blur backdrop containing Home, Explorer, Splitwise, Reports, and History.
  • Active Spring Indicator: High-end animated glowing bubble under the active page icon.
  • Apple Notch Support: Viewport rules expanded via viewport-fit=cover so content flows edge-to-edge.
  • Header Simplification: Hidden verbose desktop links, keeping only essential settings, reset, and dark mode toggles at the top.
